Detailed Description

Overview

Blink-DB Part-A is a high-performance, write-optimized database engine implemented in C++. It leverages LSM-Trees (Log-Structured Merge-Trees) as the core data structure, enabling efficient writes and optimized data retrieval. The system is designed to handle high-throughput workloads with minimal latency.

Key Features

  • Write-Major Architecture: Optimized for fast writes, ensuring quick insert operations.
  • LSM-Tree Based Storage: Efficiently manages and compacts data for optimized read and write performance.
  • Thread-Safe Execution: Supports concurrent operations using internal synchronization mechanisms.
  • Command-Based Interaction: Provides a CLI-based interface for executing database operations.
  • Persistence Support: Data is structured for efficient disk storage and retrieval.

Compilation and Execution

The following make commands are available:

Run the Database Engine

make run
  • Compiles src/main.cpp with C++17 standard.
  • Executes the database engine.

Generate Documentation

make docs
  • Generates documentation using Doxygen.

Clean Up Binaries

make prune
  • Removes compiled binaries and data files.

Supported Commands

Data Operations

Set a key-value pair

SET <key> <value>

Stores the key-value pair in the database.

Retrieve a value

GET <key>

Fetches the value associated with the given key.

Delete a key

DEL <key>

Removes the key-value pair from the database.

Utility Commands

Show available commands

help

Displays the list of supported commands.

Clear the screen

clear

Clears the CLI screen.

Exit the application

exit

Terminates the Blink-DB CLI session.

Example Usage

SET name "BlinkDB"
GET name
DEL name
GET name # Returns NULL

Additional Notes

  • Blink-DB Part-A is designed for high-speed writes, making it ideal for write-heavy workloads.
  • Reads are optimized through efficient indexing and LSM-Tree compaction.
  • Future versions will include additional features such as range queries, snapshots, and replication.

For more details, refer to the documentation in the docs/ directory.
