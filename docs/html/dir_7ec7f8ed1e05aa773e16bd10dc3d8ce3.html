partb Directory Reference

Detailed Description

Overview

Blink-DB Part-B is a high-performance, non-blocking server built using kqueue for event-driven I/O. It processes Redis Serialization Protocol (RESP) commands and interacts with the LSM-tree storage engine from Part-A.

Project Structure

.
├── engine -> ../../parta/src/engine
├── kqueue_server.hpp
├── main.cpp
└── resp
├── resp_decoder.hpp
└── resp_encoder.hpp

Features

  • Non-blocking kqueue-based server
  • RESP command processing (GET, SET, DEL)
  • Optimized for fast writes using LSM-trees
  • Benchmarking support using redis-benchmark

Build & Run

Build and Execute the Server

make run

Benchmarking

To benchmark the server performance:

make benchmark

This runs redis-benchmark with different loads and stores results in the result/ directory. The benchmarking script tests with:

Data Size Key Size Concurrent Requests Parallel Connections
512 bytes 1024 10,000, 100,000, 1,000,000 10, 100, 1000
